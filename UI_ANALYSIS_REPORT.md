# APP UI 设计分析报告

## 1. 整体布局结构

### 1.1 整体视觉框架
这是一个「多源聚合影视播放器 / 追剧应用」的完整设计体系，以 **iPhone 竖屏移动端界面** 为标准，页面采用：

- **顶部状态栏**
- **顶部标题/搜索区**
- **内容主区域**
- **底部 TabBar 导航**

整体是典型的 **"上搜索/筛选，中内容，下导航"** 的结构。

### 1.2 信息层级方式

#### A. 首页型布局
- 顶部搜索
- 分类切换
- 大卡片推荐
- 快捷入口
- 继续观看
- 源状态监控
- 底部导航

特点是信息密度中等，强调推荐和快捷入口。

#### B. 列表型布局
- 顶部标题/搜索/筛选
- 左侧分类栏或顶部标签
- 右侧列表结果
- 适合检索、筛选、管理

#### C. 详情/播放型布局
- 上部主内容（海报/播放器）
- 下部功能控制区
- 多源切换与清晰度设置较突出

## 2. 颜色方案

### 2.1 主色
- **主色：#6366F1** (蓝紫色系/indigo)
- 用于主要按钮、选中状态、进度条、标签高亮
- 整体观感偏科技、清爽、现代

### 2.2 辅助色
- **成功色：#22C55E** - 可用、正常、已启用状态
- **警告色：#F59E0B** - 提示、热度、强调信息
- **危险色：#EF4444** - 异常、故障、失败、危险提示
- **信息色：#0EA5E9** - 信息提示、蓝色标签

### 2.3 中性色
- 页面背景：极浅灰 / 纯白
- 卡片背景：白色
- 分割线：浅灰
- 主文本：深灰/近黑
- 次文本：中灰
- 禁用态：浅灰

## 7. 文字层级和排版

### 7.1 标题层级

#### 一级标题
- 页面标题、频道名、影视名称
- 字号最大，颜色最深，通常加粗

#### 二级标题
- 分区标题，如"继续观看""热搜榜""资源推荐""排行榜"
- 视觉上比一级标题小一档

#### 三级文本
- 描述文案、副标题、进度、更新信息、来源说明
- 颜色更浅，字号更小

### 7.2 文字排版特点
- 左对齐为主
- 标题 + 副文案的双层结构
- 列表项中常见"左主右辅"布局
- 数值型信息右侧对齐或右上角显示

### 7.3 字体风格
- iOS 系统字体风格
- 中英混排清晰
- 数字信息强调明显（评分 9.2、进度 80%、响应 120ms、集数 18）

### 7.4 排版节奏
- 模块间距一致
- 卡片内部留白充足
- 标题与内容之间有明确间距
- 分割线轻而不重

## 8. 特殊 UI 元素

### 8.1 多源状态监控
- 各条源的可用性、延迟、成功率、失败状态
- 颜色表达：绿色=正常、橙色=警告、红色=危险

### 8.2 "智能切换"机制
- 自动选择可用源
- 根据速度和可用性切换

### 8.3 播放器中的多线路切换
- 源列表、清晰度、倍速、字幕、自动跳转下一集

### 8.4 迷你播放器/底部播放条
- 当前频道、播放状态、暂停/播放按钮

### 8.5 首页功能入口的彩色图标组
- 电视剧、电影、综艺、动漫、纪录片
- 圆形图标，彩色渐变/单色识别

### 8.6 VIP 卡片
- 金色渐变风格
- 视觉上非常突出
- 用于会员转化

## 9. 设计总结

### 核心特点
1. **结构清晰**: 顶部搜索 + 中间内容 + 底部导航
2. **信息密度高但不杂乱**: 通过卡片、分区、标签、颜色进行分层
3. **强工具属性**: 源管理、切源、监控、下载、历史、收藏都很完整
4. **统一设计语言**: 蓝紫主色 + 白底 + 圆角卡片 + iOS 风格
5. **交互效率高**: 强调快速筛选、快速播放、快速切源
6. **商业化入口明确**: VIP 卡片突出但不突兀

### 产品定位
这是一套完整的 **多源聚合影视播放器产品体系**，不只是播放器，更是 **资源聚合与线路管理工具**。

### 10.16 顶部导航栏类型归纳

#### 类型 A：搜索型 Header
- 页面：首页、探索页
- 结构：`[搜索框] + [历史/刷新图标]`

#### 类型 B：Tab 型 Header
- 页面：直播页、下载管理页
- 结构：`[Tab 切换] + [搜索图标]`

#### 类型 C：用户信息型 Header
- 页面：我的页
- 结构：`[用户信息] + [通知] + [设置]`

#### 类型 D：标准二级页 Header
- 页面：历史记录、收藏夹、自定义源管理、设置、播放设置、源列表
- 结构：`[返回箭头] + [标题] + [操作按钮]`

#### 类型 E：内容详情型 Header
- 页面：详情页、播放器页、集数切换页
- 结构：`[返回箭头] + [内容信息] + [分享/下载/更多]`
